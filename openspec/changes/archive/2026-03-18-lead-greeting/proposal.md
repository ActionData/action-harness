## Why

The repo-lead agent launches with generic persona instructions that tell it to greet the user, but the greeting itself is entirely improvised by the LLM at runtime. There's no structured explanation of what the lead can actually do — users discover capabilities by accident or not at all. A well-crafted greeting that explains the lead's role in the harness ecosystem and surfaces actionable suggestions will make the interactive experience immediately useful instead of requiring the human to already know what to ask for.

## What Changes

- Update the lead agent persona (`.harness/agents/lead.md`) with a richer Interactive Mode section that:
  - Explains the lead's role as the human's interface to the harness pipeline
  - Lists concrete task categories the lead can help with (implementing GitHub issues, dispatching OpenSpec changes, exploring future features, creating proposals, answering repo questions, performing analyses)
  - Provides a more detailed greeting template that demonstrates capability awareness
  - Includes guidance on how to present capabilities without being overwhelming (prioritized suggestions, not a feature dump)
- Update the greeting example to show capability-aware suggestions tied to the repo's current state

## Capabilities

### New Capabilities
- `lead-greeting`: Defines the greeting behavior, capability presentation, and suggested-task generation for the repo-lead's interactive mode

### Modified Capabilities
- `lead-interactive`: The interactive session greeting requirements change — the lead must now present its capabilities and role explanation as part of the greeting

## Impact

- `.harness/agents/lead.md` — the lead persona prompt gets an enhanced Interactive Mode section
- No code changes to `src/action_harness/lead.py` — the greeting is generated by the LLM from the persona prompt, not hardcoded
- No CLI changes — the `harness lead` command already supports interactive mode
