## ADDED Requirements

### Requirement: Open PR from worktree branch
The system SHALL open a GitHub pull request from the worktree branch to the repo's default branch. The PR title SHALL follow the format `[harness] <change-name>`. The PR body SHALL include a summary of what was implemented and a note that it was generated by action-harness. The system SHALL push the branch before creating the PR.

#### Scenario: PR creation after successful eval
- **WHEN** all eval commands pass
- **THEN** the system pushes the worktree branch and opens a PR via `gh pr create` with a structured title and body

#### Scenario: PR body content
- **WHEN** a PR is created
- **THEN** the body includes: the change name, a summary of the implementation, eval results, and a footer noting it was generated by action-harness

### Requirement: Human reviews and merges
In the bootstrap phase, the system SHALL NOT auto-merge. The PR is opened for human review. The system exits after PR creation with a message indicating the PR URL.

#### Scenario: Bootstrap merge flow
- **WHEN** the PR is created successfully
- **THEN** the system prints the PR URL and exits. A human reviews and merges the PR.
